10 Principios de Usabilidad de Jakob Nielsen

Guía fundamental para diseñar interfaces fáciles de usar

1. Visibilidad del estado del sistema

El sistema siempre debe informar al usuario lo que está sucediendo mediante retroalimentación clara.

Ejemplo: Una barra de carga cuando se sube un archivo.

2. Relación entre el sistema y el mundo real

La interfaz debe usar palabras, conceptos e iconos familiares para el usuario.

Ejemplo: Un icono de papelera para eliminar archivos.

3. Control y libertad del usuario

Los usuarios deben poder deshacer o rehacer acciones fácilmente.

Ejemplo: Botón de "Deshacer" después de eliminar un correo.

4. Consistencia y estándares

Los elementos de la interfaz deben mantener un mismo estilo y comportamiento.

Ejemplo: El mismo diseño de botones en todas las páginas.

5. Prevención de errores

El diseño debe evitar que los usuarios cometan errores.

Ejemplo: Confirmación antes de borrar un archivo.

6. Reconocimiento antes que recuerdo

Es mejor mostrar opciones visibles que obligar al usuario a recordar información.

Ejemplo: Menús desplegables con opciones disponibles.

7. Flexibilidad y eficiencia de uso

El sistema debe permitir atajos para usuarios avanzados.

Ejemplo: Usar Ctrl + C para copiar texto.

8. Estética y diseño minimalista

Las interfaces deben ser simples y no contener información innecesaria.

Ejemplo: Un diseño limpio con pocos elementos en pantalla.

9. Ayudar a los usuarios a reconocer y solucionar errores

Los mensajes de error deben explicar claramente el problema y cómo solucionarlo.

Ejemplo: "La contraseña debe tener al menos 8 caracteres".

10. Ayuda y documentación

El sistema debe proporcionar ayuda accesible cuando el usuario la necesite.

Ejemplo: Sección de preguntas frecuentes (FAQ).
